        # 谈谈为啥需要github和个人博客

        导语: 有一个多星期没有写过技术类的博客了, 最近忙着招人所以谈谈在筛选简历的时候一些心得吧。

        # 起始

        在之前的文章作为面试官谈谈我对面试的理解有介绍过如何编写一个简单的简历；其中有一个点就是可以在简历中放置自己的博客或者 github 地址。

        谈谈为啥有自己的博客或者 github 吧。

        从求职者自身角度而言:

        • 能够通过博客总结日常所遇到的问题以及解决方案;
        • 能够通过博客让更多的同行认识自己,提高自己的知名度;
        • 通过知识的沉淀,也为互联网的发展贡献自己的一份力量;
        • 如果自己的博客用户量达到一定的数量,会有广告商找你合作哦;
        • 当然如果 github 上有高星项目，也是简历上浓墨重彩的一笔;

        从面试官的角度

        讲真,每次筛选/查看简历的时候,遇到有贴自己博客或者 github 地址的同学，我都会点击进去查阅一番,当然博客也分好坏，但是能有此行动:

        • 此同学是是一个乐于总结的人;
        • 若文章写的比较深入, 从某种角度可以反映出此同学对某种技术的了解程度;
        • 若 github 上有高赞项目,从侧面可以反映出此同学热爱开源,对某类技术一定的深入了解;能够解决当前行业的某一类问题;

        从其他角度

        • CSDN 广告太多,简书/掘金/公众号有限制，生怕开发者引流;
        • 等

        # 一定需要博客吗?

        一定需要博客/github 吗? 我想这不一定。

        在查阅很多简历发现，有许多同学贴了 github/博客地址;查阅后发现部分同学有一些地方有待改进（个人理解）:

        • 建议博客分类清楚,方便查阅;
        • 建议选择 markdown 编辑器/展示时，内容配色及代码部分能够清晰展示;
        • 文章建议添加目录，可以点击跳转;
        • 文章内容建议不要过于简单，希望能体现出个人对某种问题的思考和记录;
        • 文章内容建议图文并茂/代码及思考并茂;
        • 建议博客不要强加不协调动画;
        • github 上建议将自己认为比较满意的项目放置首页;

        # 哪些写博客的工具

        下面简单介绍一下笔者(mrgaogang)之前使用过的博客工具,此处只简单介绍一下使用各类博客工具展示的大致效果，其具体搭建详情请见这里

        # Vuepress(推荐)

        特点:

        • 界面简洁优雅
        • 容易上手（半小时能搭好整个项目）
        • 更好的兼容、扩展 Markdown 语法
        • 响应式布局，PC 端、手机端
        • Google Analytics 集成
        • 支持 PWA
        • 支持使用 vue 组件编写修改页面

        # Hexo(推荐)

        特点:

        • 快速、简洁且高效的博客框架
        • 只需一条指令即可部署到 GitHub Pages, Heroku 或其他平台。
        • Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。
        • 强大的 API 带来无限的可能，与数种模板引擎（EJS，Pug，Nunjucks）和工具（Babel，PostCSS，Less/Sass）轻易集成

        # hugo

        特点:

        • 极快的页面编译生成速度。（ ~1 ms 每页面）
        • 完全跨平台支持，可以运行在 Mac OS X, Linux, Windows, 以及更多!
        • 安装方便 Installation
        • 本地调试 Usage 时通过 LiveReload 自动即时刷新页面。
        • 完全的皮肤支持。
        • 可以部署在任何的支持 HTTP 的服务器上。

        # gitbook

        优点：

        • 页面简洁
        • 支持多人协作写作
        • 支持本地图片插
        • 支持 build 成静态网页，自己部署方便

        缺点：

        • 提供的编辑器使用 git 来记录，每次 save 都要 commit 一次，commit 太多记录
        • 要翻墙才能访问他们的网站

        # 自己搭建博客

        自己搭建博客需要考虑的点比较多:

        • 使用什么框架？
        • 是否支持富文本？Markdown?
        • 图片存在哪里?
        • 页面的整体布局?
        • 国际化？
        • 响应式？
        • 博客的自动化部署？
        • 域名绑定？

        如果是新同学，或者对前端还不太熟悉的话，建议自己搭建博客，可以从中学习到很多新的知识。
